# MVC
* Model View controller
* 장점
    * 패턴들을 구분해 개발한다 
    * 유지보수가 용이하다 
    * 유연성이 높다 
    * 확장성이 높다 
    * 협업에 용이하다 
    * 코드의 가독성 높아진다 
* 단점 
    * 완벽한 의존성 분리가 어렵다
    * 설계 단계가 복잡하다 
    * 설계시간이 오래 걸린다 

### Controller 
view와 Model을 연결해주는 부분 

### Veiw 
UI 관련된 것을 처리하는 부분(사용자에게 보여지는 부분)

### Model 
데이터베이스와 연결되어 데이터를 처리하는 부분
